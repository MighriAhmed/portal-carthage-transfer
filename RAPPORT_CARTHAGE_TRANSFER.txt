MODÈLES DE DONNÉES
===============================================================

1. USER MODEL
- role: 'admin' ou 'supplier'
- supplierName: Nom du fournisseur
- username: Nom d'utilisateur
- email: Email
- phoneNumber: Téléphone
- Méthodes: isAdmin, isSupplier

2. BOOKING MODEL
- id: ID unique
- bookingId: ID réservation client
- status: Statut (Pending, Confirmed, Completed, Canceled)
- pickupDateTime: Date/heure prise en charge
- addresses: Liste des adresses
- supplier: Fournisseur assigné
- price: Prix total
- earning: Gains fournisseur
- passengersNumber: Nombre passagers
- firstName, lastName: Nom client
- phoneNumber: Téléphone client
- driverName: Nom chauffeur
- plateNumber: Plaque véhicule
- Méthodes: fullName, route, isGoogleAds

PROVIDERS (GESTION D'ÉTAT)
===============================================================

1. AUTH PROVIDER
Responsabilités:
- Authentification (login/logout)
- Stockage sécurisé JWT
- Vérification tokens
- Headers autorisation

Méthodes principales:
- login(username, password)
- logout()
- authHeaders

2. BOOKING PROVIDER
Responsabilités:
- CRUD réservations
- Gestion fournisseurs
- Filtrage/recherche
- Communication API

Méthodes principales:
- fetchBookings()
- fetchLatestBookings()
- updateBooking()
- addBooking()
- deleteBooking()
- addSupplier()
